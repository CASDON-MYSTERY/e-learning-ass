TREE

Tree is a discrete structure that represents hierarchical relationships between individual elements or nodes or vertices, with the constraints that no reference is duplicated, and none points to the root or having any closed loops (cycles). It is a connected acyclic undirected graph. There is one and only one path between every pair of vertices in a tree.

BASIC TREE TERMS

  1. The first node from where the tree originates is called as a root node. In any tree, there must be only one root node. We can never have multiple root nodes in a tree data structure. Node F is the root node in the diagram above.
  2. The connecting link between any two nodes is called as an edge . In a tree with n number of nodes, there are exactly (n-1) number of edges.
  3. The node which has a branch from it to any other node is called as a parent node . In other words, the node which has one or more children is called as a parent node. In a tree, a parent node can have any number of child nodes. F,B,G,D,I are parents nodes in the diagram above.
  4. The node which does not have any child is called as a leaf node . Leaf nodes are also called as external nodes or terminal nodes . Leaf nodes in the diagram above are A,C,E,H.
  5. The node which has at least one child is called as an internal node . Internal nodes are also called as non-terminal nodes . Every non-leaf node is an internal node. F,B,G,D,I are internal nodes in the diagram above.
  6. The node which is a descendant of some node is called as a child node . All the nodes except root node are child nodes. B,G,A,D,I,C,E,H are child nodes in the diagram above.
  7. Nodes which belong to the same parent are called as siblings . In other words, nodes with the same parent are sibling nodes. B and G, A and D, C and E are siblings in the above diagram.
  8. In a tree, each child from a node forms a subtree recursively. Every child node forms a subtree on its parent node.
